# CAN 2026 Poster — Reproducible Figure Plan

## Project Overview

Poster presenting work on **postpartum depression (PPD)** using two spatial transcriptomics datasets:
- **Slide-tags** — single-cell resolution spatial transcriptomics
- **Slide-seq** — spatial transcriptomics for validation

All figures will be output as **SVG** for use in a graphic design tool (e.g., Illustrator, Inkscape, Figma).
All figure scripts will be written in **Python**.

---

## Repository Structure

```
CAN-2026-poster/
├── data/
│   ├── raw/           # Large source files (h5ad, etc.) — gitignored
│   └── processed/     # Extracted CSVs for each figure — gitignored if large
├── figures/           # Output SVGs — gitignored (regenerated by scripts)
├── scripts/
│   ├── extract/       # One script per dataset to extract CSVs from raw files
│   ├── figures/       # One script per figure (fig01_umap.py, etc.)
│   └── run_all.py     # Wrapper: runs all extract → figure scripts in order
├── docs/
│   └── plan.md        # This file
├── CLAUDE.md
└── .gitignore
```

---

## Workflow

```
Raw data files (h5ad, etc.)
        |
   [scripts/extract/]      ← Run once per dataset to generate CSVs
        |
  data/processed/*.csv
        |
   [scripts/figures/]      ← One script per figure, reads only from data/processed/
        |
    figures/*.svg
        |
   [run_all.py]            ← Runs the full pipeline end-to-end
```

**Key principle**: figure scripts must only read from `data/processed/`. No figure script should load a raw `.h5ad` directly. This keeps figure iteration fast and decoupled from heavy data loading.
